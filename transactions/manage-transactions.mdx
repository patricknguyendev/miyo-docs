The **Transactions** page is the source list behind your spending summaries, recurring detection, and category totals.

## Find a transaction

Use search and filters to narrow the list by account, category, or transaction type. Pagination keeps longer histories manageable.

## Add or edit a transaction

You can create a manual transaction from the header add menu or the Transactions page. When you edit a transaction, you can update details such as its category and notes.

Miyo also supports metadata for excluding a transaction from budgets and marking eligible records as tax-deductible.

## Categorize activity

Assign a category when a transaction is uncategorized or when the suggested category is not useful. Your category choices affect spending and budget views.

<Note>
  Transactions can come from manual entry, a connected account, an import, automatic categorization, or a transaction rule. Review important records rather than assuming every source is complete.
</Note>

## Remove a transaction

Deleting a transaction removes it from normal views without immediately destroying the underlying record. If a connected institution sends the transaction again, its source data can affect what happens next.
